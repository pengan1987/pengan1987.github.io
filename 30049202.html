Palm简史（五）大结局

作者：蓬岸 Dr.Quest
知乎文章编号：30049202
创建于：2017-10-13 12:28:01
修改于：2017-10-13 8:24:25

Palm最开始是一家软件公司，但由着创始人独到地远见，他们走向了PDA制造商的道路。Palm曾经是PDA世界的王者，亦是智能手机市场上的一支劲旅。他们到底做错了什么？

迷途的Palm

简而言之，Palm在开始其PDA业务的时候，他们走在一条正确的道路上，因为他们优秀的Graffiti手写输入法在其他设备上工作的并不那么完善。除此之外，相比杰夫·霍金斯（Jeff Hawkins）所期望的便携式设备相比，早期的PDA都过于巨大了。

霍金斯完成了他的愿望，初出茅庐的PalmPilot在1996年重新定义了整个PDA行业。在1997年，他们推出了Pilot的改进型号和Palm OS 2.0，接下来又在1998年推出了Palm III和Palm OS 3.0。

然后Palm原地踏步了一段时间，直到2001年3月Palm OS 4.0才开始出货。然后是2003年10月和Palm OS 5一同登场的Palm Tungsten系列，与此同时也完成了从摩托罗拉龙珠（DragonBall ）CPU向ARM的迁移，并在此期间将PalmSource（负责开发Palm OS）和Palm（负责开发硬件）一分为二，我们将在后面详细讨论。

Handspring

1998年，Palm推出Palm III和Palm OS 3.0之后，大部分的创始人都离开Palm去到新成立的Handspring，这家新公司将会持有Palm OS的授权，并生产与Palm不同的产品。假设这批创始人如果当时留在Palm，也许Palm OS 4.0就会在1999年，最晚不超过2000年上市。

带有实体键盘的Treo 180和使用Graffiti手写区的Treo 180g

1999年，我们看到Handspring Visor的出现，它使用了改进后的Palm OS 3.1，虽然此时Palm也在缓慢地改进其硬件，但Handspring却发明了他们的新款智能手机，即2002年出现的Treo 180。

Palm收购BeOS

1990年，让-路易·加西（Jean Louis Gassée）怀着创造次世代操作系统的梦想离开曾经工作过的苹果公司，带着一群追随他的苹果雇员成立了Be Inc.

BeOS专门为多处理器环境、普适多线程（pervasive multithreading）和抢占式多任务（preemptive multitasking）进行优化。它从设计之初就被定位于一款多媒体操作系统，无论电脑在运行什么任务，声音和视频操作都不会被打断。

BeOS最初是为双处理器电脑所设计的，Be计划中使用一对AT&T Hobbit来开发他们的电脑系统。不过后来他们选择了PowerPC 603作为BeBox的CPU。1996年10月推出的BeBox运行频率为66Mhz，1997年8月这一速度则被提升为133Mhz。

BeOS后来被移植到Mac电脑上，而BeBox则在1998年停产，Be公司决定将注意力集中到BeOS操作系统本身和相关的软件商。就已知的资料而言，虽然BeOS的软件数量很少，但优秀的稳定性和适应性仍然成为BeOS的亮点。1997年，Power Computing决定购买BeOS授权并和自家的Mac兼容机捆绑销售。我还了解到摩托罗拉也试图为StarMax兼容机购买BeOS授权，但这笔买卖并没有成交。

同样在1997年，苹果成为寻找一款新的操作系统来替代经典版Mac OS。苹果的Copland计划已然宣告破产，而寻找替代品的目光则落到BeOS和NeXTstep身上——这两家公司都是由曾经的苹果高层成立。苹果对BeOS很感兴趣，但认为让-路易·加西要价实在太高了。最后苹果决定收购NeXT，史蒂夫·乔布斯也回到了苹果公司，Mac OS X也在这时浮出水面。

而Be和苹果之间的关系也一落千丈，1998年，BeOS被移植到英特尔x86架构，Mac版的BeOS则停止更新，无法支持PowerPC G3及更新的CPU。但无论怎样，Be都无法吸引到足够的人来支持BeOS的开发，最终在2001年，Be被Palm收购。

当2003年10月Palm将其硬件和操作系统部门永久性的分开时，BeOS和Palm OS被一同划分给了PalmSource。此时的PalmSource希望将BeOS作为下一代操作系统Palm OS 6的基础。

PalmSource：ARM处理器和新的风险

协调理论上已经一分为二的硬件和软件公司是一件颇为挑战的事情，Palm发现最高主频只有33Mhz的龙珠处理器已经难以维持其在市场上的竞争力，他们决定将硬件迁移到ARM处理器。而Palm硬件平台的转换意味着PalmSource不得不为ARM处理器移植或重写Palm OS。

从技术的角度讲，最初的Palm OS是以AMX实时操作系统（RTOS）为核心的。用作Palm OS 1~4内核的Kadak AMX RTOS可以支持几种不同的处理器——英特尔8086和386/486（原文为384/466应为原作者笔误），PowerPC，MIPS和摩托罗拉68000。

正如Thom Holwerda在osNews网站上所述的一样，Palm修改了AMX 68000系统，删除了所有被认为是不必要的功能，抢占式多任务正是被删除的功能之一。Palm从来不打算让运行在龙珠处理器上的Palm OS成为一款多任务操作系统，而这也是Palm OS 1~4响应速度飞快的原因——它同一时刻只能运行一个程序。

当Palm计划将Palm OS 5迁移到到ARM平台时，他们本来可以选择使用AMX 4-ARM内核，但最终还是决定自行开发新的内核。新的内核提供了与旧内核“基本相同的功能”，并留下了进一步扩充功能的希望。Palm OS 5登陆的第一款机型，是2002年11月的Palm Tungsten T。

没错，Palm OS 6确实出现过

PalmSource宣布他们计划在2003年底发布Palm OS 6 Cobalt（钴），但直到2004年1月才正式发布，Cobalt吸收了BeOS的许多功能。虽然可以被所有Palm OS授权持有者使用，但却没有一款使用Palm OS 6的产品真正在市场上出现过。

PalmSource在2002年1月正式成为Palm的子公司，在2003年完全独立。

创始人回来了！

这段时间里发生的最好的事情莫过于Palm在创始人出走成立Handspring之后，两家公司随着2003年6月的合并案合二为一。随之带回了杰夫·霍金斯，唐娜·杜宾斯基（Donna Dubinsky）和艾德·柯林根（Ed Colligan）——PalmPilot成功背后的三人组合，他们三人在1998年6月离开（作为3Com一个部门的）Palm并成立了Handspring。

不幸的是，创始人的回归已经太晚了，让我们看一看这段时间内Palm所发生的大事件：

  • 1998年3月。Palm OS 3发布。
  • 1998年7月，霍金斯，杜宾斯基和柯林根离开了Palm。
  • 2001年3月，Palm OS 4在Palm OS 3推出三年之后发布。
  • 2001年7月：Palm公开宣布转向ARM处理器的计划。
  • 2001年8月：PalmSource收购BeOS。
  • 2002年1月：PalmSource成为Palm的子公司。
  • 2002年11月：推出基于ARM的Palm Tungsten T，使用了144 MHz TI OMAP ARM处理器和Palm OS 5。
  • 2003年6月：创始人回到了Palm。
  • 2003年10月：PalmSource独立运作。
  • 2004年1月：PalmSource发布Palm OS 6 Cobalt。 但没有任何厂商采用。

当创始人归来之时，不好的预兆也已经开始：Palm已经将硬件和操作系统部门分离，而PalmSource正在努力争取在2003年底发布Palm OS 6。

被索尼扼杀的Clié

索尼在2000年开始成为Palm授权厂商之一，其Clié系列PDA以多媒体应用为主要特色。2004年夏天，索尼开始从市场上撤离，仅仅在日本继续销售Clié产品，到2005年春天，索尼彻底关上了Palm OS和Clié的大门。

关于Clié产品线的介绍，请参阅2015年2月24号Damien McFerran为Know Your Mobile网站撰写的文章：Looking Back At Sony's CLIÉ Series: An iPhone For An iPhone-Less World。

从PDA到智能手机

Palm将重点从PDA转向智能手机，这看起来将会是一个更大的市场。Palm的最后两款PDA是99美元的Palm Z22和顶级的Palm TX，这两款机型都发布于2005年10月。接下来所有的产品都只有智能手机，但这并没有让事情带来转机，PDA市场萎缩的速度和智能手机市场的增长一样快。

事实上，如果不是Treo智能手机的支撑，Palm可能早在2005年之前就关门了。

Treo上的Windows？

2005年9月，Palm做了一件令所有人都意想不到的事情，它发布了一款没有运行Palm OS的设备。具体来说，是一款使用Windows Mobile的Palm。

我想他们一定是太想证明自己是一家硬件公司，而不会在Palm OS一棵树上吊死。我不清楚他们将如何向Palm OS用户交代，但我想Palm OS用户们一定不会答应。

CDMA版本的Palm 700w在2006年1月上市，使用了Windows Mobile 5.0和更低分辨率的240x240像素屏幕——因为Windows无法支持更高的320x320分辨率，700w在美国通过Verizon独家发售，而在英国则是由Vodaphone独占。

Palm还制造了Treo 750v，这是一款仅在欧洲销售的Windows Mobile 5.0设备。

2006年10月，Palm推出了几乎与700w完全相同的CDMA手机Treo 700wx，但这款机型不再由Verizon独占。

接下来的一款Windows Mobile型号则是Treo 500v，它的尺寸和样式与Palm Centro几乎相同，但使用了320x240像素显示屏，运行Windows Mobile 6.0的Treo 500v从2008年2月开始发货。

第四款Windows Mobile Treo是2008年中期的800w，这款机型是美国运营商Sprint独占的CDMA机型，也是第一款使用320x320显示屏的Windows Palm，它使用了Windows Mobile 6.0系统。

2009年1月5号，Palm发布了Treo Pro，这款机型有GSM和CDMA两个版本，运行Windows Mobile 6.1。

销售使用Windows Mobile系统的Treo算不算是个好主意？这可能是个永远都无解的问题，因为2009年6月Palm又一次更换了新的操作系统。

Palm Pre和webOS

新推出的Palm Pre和webOS，从硬件到软件都打破了Palm以往的传统。

Palm Pre是Palm第一款滑盖手机。除非是在滑开的状态下，不然键盘不会出现在我们的视线当中。由于采用了隐藏式键盘，它看起来与iPhone更加相似——它还有着与早期iPhone相同的320x480像素屏幕，只是尺寸稍小一些，只有3.1英寸，而iPhone则是3.5英寸。这也让Pre相比iPhone 3G要稍小一些，许多人认为Pre是苹果iPhone遇到的第一个真正的竞争对手。

基础型号的Pre有8GB存储空间，而后来的Pre Plus则增加到16GB。两款机型都没有内存卡插槽。

一开始Pre是Sprint的独占机型，并成为Sprint有史以来最受欢迎的智能手机。预计在6月到8月这一个季度间Sprint销售了50万台Palm，以及32万5千台Palm OS 5智能手机。

Palm Pre支持被称作“点金石(Touchstone)”无线充电技术

Pre是第一款使用webOS的移动设备，这是由Palm开发的新款操作系统（而不是PalmSource，在Palm OS 6失败后，PalmSource在2005年被爱可信收购）。Palm OS 1~4使用AMX RTOS作为其内核，Palm OS 5使用了Palm自己为ARM处理器开发的内核，Palm OS 6 Cobalt借用了BeOS的技术，而这一次Palm则在Linux的基础上开发webOS。（这听起来有些熟悉，因为Android也是在Linux的基础上构建的）。

与BeOS一样，webOS从一开始就支持多任务（与之相比iPhone只能有一个活跃的应用程序）。

webOS曾经推出过三个主要版本：

  • webOS 1推出于2009年，和Pre一同发布，在Pixi发布时升级为1.2.9.1版，最后一版则是1.4.5.1
  • webOS 2推出于2010年，和Pre 2一同发布，并随着HP Veer的推出升级到2.1.1版，和HP Pre 3一同升级到2.2版，最后一个版本则是2.2.4
  • webOS 3在2011年和HP TouchPad一同推出，最后一版是2012年1月12号的3.0.5版，同时也被用在LG的智能电视上。

Palm的第二款webOS智能手机叫做Pixi，它具有糖果般的圆润外形，但没有使用滑盖键盘。在2009年11月推出时是当时市面上最小的智能手机。

2010年4月28日，惠普宣布了其收购Palm的计划。并在2010年7月1日完成收购。之后惠普开始着手替换原有的品牌，Pixi（的后续机型）在2011年更名为HP Veer，Pre 3也被贴上惠普商标，此时的惠普曾对webOS踌躇满志。

2011年7月，惠普发布了其TouchPad平板电脑，它使用了9.7英寸1024x768像素显示屏和webOS 3.0，它和主要的竞争对手苹果iPad 2有着相同的屏幕尺寸和分辨率。而不带手机数据网络支持的版本价格也向iPad看齐：499美元16GB，599美元32GB，699美元64GB。相比iPad 2，TouchPad有着更快的CPU和两倍容量的主内存。

webOS智能手机和平板电脑

  • Palm Pre：500 MHz ARM Cortex 8+处理器，3.1英寸显示屏，320万像素摄像头，256 MB RAM，8 GB存储空间，802.11g WiFi，webOS 1.0.2，上市时间：2009.06.06
  • Palm Pixi：主要规格与Pre相近，但使用了2.6英寸320 x 400像素显示屏，200万像素摄像头，并且没有WiFi，上市时间：2009.11.15
  • Palm Pixi Plus：与Pixi相同，但加入了802.11g WiFi
  • Palm Pre Plus：512MB RAM，16 GB存储空间，webOS系统，上市时间：2010.01.25
  • Palm Pre 2：1 GHz处理器，500万像素摄像头，webOS 2.0，Flash Player 10.1 Beta，这款机型于2010年10月22日在法国首发。
  • HP Veer：800 MHz高通Scorpion（蝎子）处理器，500万像素摄像头，8 GB存储空间，webOS 2.1.2，发布时间：2011.05.15，2年合约价99美元。
  • HP TouchPad：1.2 GHz或1.5 GHz双核Scorpion处理器，1 GB RAM，16/32/64 GB存储空间，9.7英寸1024 x 768屏幕，webOS 3.0，上市时间2011.07.01
  • HP Pre 3：1.4 GHz 高通Snapdragon（枭龙）处理器，3.6英寸显示屏，8GB或16 GB存储空间，802.11n WiFi，webOS 2.2.4，2011.08.17在英国首发。
  • 惠普TouchPad Go：1.5GHz双核Scorpion处理器，32GB存储空间，7英寸显示器，但从未正式发布。

不过webOS从未真正获得过成功，它是一款优秀的操作系统，但在LG将其用在智能电视上面之前，一直都只有一家公司在使用webOS。

2011年8月18日，HP宣布将停产所有的webOS设备，包括前一天刚刚发布的Pre 3。

2011年10月22日，HP将Android 2.3 Gingerbread移植到TouchPad，用户将可以在开机时选择使用哪款操作系统。2012年1月17号，beta版本的Android 4.0 Ice Cream固件也被移植到TouchPad。

在被HP放弃之后，webOS被捐赠给开源社区，成为Open webOS。

谁杀死了Palm？

一连串的错误决策才是杀死Palm的元凶：3Com迟迟不肯让Palm独立运作，Handspring的出现消耗了Palm的领导力并扰乱了业务的方向。对外许可Palm OS实际上给Palm带来了不少好处，但随着PalmSource单独运作，新一代的Palm OS 6 Cobalt失去了其用武之地。

webOS的卡片式多任务切换后来被Android和iOS所借鉴

成为硬件公司的Palm开发了webOS，并且凭着对webOS未来的自信不再继续生产Palm OS设备。但实际上Palm很可能需要新的Palm OS设备来支撑其运作，因为在2006年的收入巅峰之后，从2007年中开始Palm的收入一直在不断下降。

到临近Pre发布的时候，Palm已经是大厦将倾了，不仅收入状况不佳，而且在2007年、2008年的多数时候，以及2009年全年都处于亏损状态。如果HP没有在在2010年收购Palm的话，Palm的寿命可能还会更短。但就算是财大气粗的HP在webOS上依旧损失惨重，2011年8月惠普放弃了其最后的努力，不得不割肉止损。

Palm将其全部赌注都压在webOS和Pre上，但最终输掉了这场赌局。

安息吧Palm，那些伟大的产品永远活在发烧友们的心中。
